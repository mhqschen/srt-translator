SRT 字幕翻译完整流程指南

拿到英文 SRT（其他语言也可以）后，做出稳定可交付的中文字幕；想追求更高级的效果，也给你一条“升级路线”。

先把软件跑起来（Python 版）

  1. 安装 Python 3.8+（安装时建议勾选 “Add Python to PATH”）。
  2. 在本项目目录打开命令行，安装依赖：
    python -m pip install -r requirements.txt
  3. 启动方式二选一：
    • 双击 run_gui.bat（更适合小白）
    • 命令行运行 python srt_gui.py
提醒：srt_gui_config.json 通常包含 API Key 等敏感信息，请自行妥善保管，避免上传到公开平台。

普通流程（推荐：简单、够用、稳定）

  1. 字幕翻译器：选输入/输出文件，选 API，建议先点一次“测试API连接”，再点击“开始翻译”。
  2. 自动跳转到字幕校验器（必须做）：翻译完成后，软件会自动切换到“字幕校验器”，并自动填好路径。
  3. 开始校验：点击“开始校验”，检查条目编号/数量/结构是否完整。
  4. 发现缺号/不匹配：直接点“一键重翻”，修好后再校验一次，直到通过。
为啥要“强制校验”？字幕最怕的不是翻得一般，而是缺条目/编号错：播放器、剪辑软件会直接乱套。先把结构保住，后面再谈质量升级。

高级流程（追求质量：纠错 → 审核 → 翻译 → 强制校验 → 润色 → 再审核 → 细修 → 双语）

  1. 字幕纠错器（先处理“听错字”）：先跑一次纠错，重点修正 ASR 常见错词/同音词，让后续翻译更省心。
  2. 自动跳转到“纠错/润色审核”（逐条看一眼）：系统会跳到审核页。你可以选择“保持原文 / 手动编辑”，改完记得保存修改。
  3. 字幕翻译器：用审核后的字幕去翻译，输出中文字幕。
  4. 自动跳转到字幕校验器（必须）：翻译完成后自动切到校验器，点“开始校验”。
  5. 缺号/不匹配就一键重翻：修复后再次校验，直到通过。
  6. 字幕润色器（让中文更像人话）：对中文译文做润色，提升口语感/一致性/可读性。
  7. 润色后再次自动跳转到“纠错/润色审核”：对润色结果再审查一次。不满意的地方同样可以“保持原文”或“手动编辑”，然后保存。
  8. 专业软件细修（可选）：Subtitle Edit 等做断行、标点、风格统一、时间轴微调、阅读速度优化等细节。
  9. 双语转换器（可选）：用“英文原字幕 + 润色后的中文字幕”合并成双语 SRT。若两份 SRT 时间轴/条目不一致，合并时将以中文 SRT为准。

推荐参数（速度 & 质量兼顾）

字幕翻译器：批次 30 / 上下文 5 / 线程 20 / 温度 0.8
不稳定时：批次 10 / 上下文 2 / 线程 5
小贴士：不稳定时别硬扛，先降线程数，再降批次大小。跑稳了，效率自然就上来了。
